בקשות קטנות וממוקדות = הצלחה.

✅ מה עובד מעולה בטלגרם

  • שנה צבע כפתור — "שנה את הכפתור האדום לכחול"
  • תקן טקסט — "שנה את הכותרת מ-X ל-Y"
  • הוסף כפתור — "הוסף כפתור 'העתק' ליד הכפתור הקיים"
  • תקן באג — "הכותרת נחתכת במובייל, תתקן"
  • שנה גודל פונט — "הגדל את הכותרת ל-18px"

הכלל: בקשה אחת, שינוי אחד, קובץ אחד

ככל שהבקשה יותר ספציפית, הסיכוי לשבירה קרוב לאפס.

⚠️ מה יכול לשבור דברים

  • "שנה את כל העיצוב" — יכול לדרוס CSS שלם
  • "תבנה פיצ'ר חדש עם 4 דברים" — יותר מדי בפעם אחת
  • "תעשה את זה כמו באתר X" — הבוט לא רואה אתרים אחרים
  • "תשנה את הפיד ואת הנרטיב ואת הכותרות" — 3 שינויים = 3 הודעות נפרדות

אם צריך שינוי גדול — עדיף לעבוד עם Claude Code (המחשב)

שינויים גדולים דורשים הבנת הקשר רחבה. Claude Code רואה את כל הפרויקט ויודע מה לא לגעת.

📝 דוגמאות נכון / לא נכון

  • "תוסיף 5 פיצ'רים חדשים"
  • "תעשה את זה כמו סטטיסטה"

🎯 איך לבקש שינוי בטלגרם

1

תאר את השינוי בצורה ספציפית

"בדף הכותרות, שנה את צבע הכפתור 'נרטיב' מאדום לסגול"

2

ציין באיזו אפליקציה

"באפליקציית לידרפיד (news-generator)" או "באפליקציית המכרזים (tenders-app)"

3

בקשה אחת בכל הודעה

אם יש 3 שינויים — שלח 3 הודעות נפרדות. ככה כל שינוי נבדק בנפרד.

4

בדוק אחרי כל שינוי

אחרי שהבוט מסיים — רענן את האפליקציה ובדוק שהכל תקין לפני שמבקשים שינוי נוסף.

🔧 מה קורה מאחורי הקלעים

כשאתה שולח בקשה לבוט, הוא:

1. קורא את CLAUDE.md — כללים שמגנים על העיצוב

2. עושה git pull — מושך את הגרסה הכי חדשה

3. קורא את הקובץ לפני שמשנה אותו

4. עורך רק את החלק הרלוונטי — לא משכתב הכל

5. בודק build — שהקוד תקין

6. עושה deploy + git commit — שומר ומעלה
